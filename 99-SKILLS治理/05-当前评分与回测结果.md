# 当前 SKILLS 评分与回测结果

评分日期：2026-04-23

## 评分口径说明

从本轮开始，本地评分不再使用“单一结构总分”，改为三段分：

- **结构分（40）**：标准目录、`SKILL.md`、frontmatter、`agents/openai.yaml`、`references/`、`evals/`、版本与治理信息。
- **实战分（40）**：触发边界、正负例、criteria 质量、行为闭环、相邻 Skill 路由边界、经验回写要求。
- **漂移分（20）**：回归/边界 eval、drift 策略、版本与 change log、按周期复测意识。

注意：当前“实战分”仍然主要是 **operational readiness（实战就绪度）**，不是线上真实成功率。真实命中率、真实完成率、周期漂移统计还需要后续单独沉淀。

#### 5.5 工程层教训 (留给以后)

**Windows stdout 重定向 + 中文字符串污染 JSON 文件**

现象:`python score_skills.py > metrics/x.json` 在 Git Bash on Windows 上,
JSON 文件实际写入了 mojibake (`ƫ�ʼǻ�` 而不是 `偏笔记化`)。

诊断绕了一圈,真正的根因:Python `sys.stdout` 在 Windows 上默认编码是 `cp936/gbk`,
`sys.stdout.reconfigure(encoding="utf-8")` 对 **shell 已经重定向** 的 stdout 不可靠。
Python 内部字符串正确,但 `print()` 输出的字节经过 GBK encoder。

**正确做法**: 给脚本加显式 `--output FILE` 参数,直接 `Path.write_text(payload, encoding="utf-8")`,
不依赖 stdout 重定向。

后续涉及多语言输出的所有 CLI 脚本都应该走这套约定 (`tools/` 下 8 个 recorder/replayer
脚本已经全部用 `Path.write_text(... encoding="utf-8")`,这次只是 score_skills.py 漏了)。


- 编码 AGENT 仍需独立沉淀，不能把当前 SKILLS 当成代码工程规则系统。

### 6.8 hook governance 上下文识别(v0.4.0 末段热修)

v0.4.0 主体收尾跑 Stop hook 时,本次治理任务自身把 hook 触翻了:

| 报错 | 原因 | 类型 |
|---|---|---|
| `检测到未沉淀的 domain: ananewsite.com, some-real-site.com` | transcript 里 eval/子 agent 的 placeholder 域名 | 真 false positive |
| `[硬检查] 域 thaiairways.com 未见新写入` | 05 文档反复引用 thaiairways 作为试点案例,被 mtime 硬检查抓 | 语境误报 |

根因:`post_task_reminder.py` 把"transcript 出现 domain + 没看到 PERSIST_MARKERS"
当 reverse 任务,但治理任务的"沉淀"落点是 `metrics/scores-v*.json` 与
`99-SKILLS治理/`,不在站点经验库。

**修复方案**(`tools/post_task_reminder.py`):

```python
GOVERNANCE_MARKERS = (
    "score_skills.py", "99-SKILLS治理", "metrics/scores-",
    "skills-evaluation-governance", "drift-history.md",
    "verify_delivery.py", "评分回测", "评分漂移",
    "Skill Bench", "scores-v0.",
)

def is_governance_task(text: str, threshold: int = 2) -> bool:
    """命中 >=2 个 marker 视为治理任务。阈值 2 排除单次引用。"""
    hits = sum(1 for k in GOVERNANCE_MARKERS if k.lower() in text.lower())
    return hits >= threshold
```

main 里:`if reverse_hit and not governance_hit:` 才进入 domain 提示与硬检查
分支;completion 自评提示不受影响(治理任务也该自评)。stats 加
`governance_hit` 字段供后续校准。

**验证**:重跑改后 hook 对本次卡死的 transcript:

```
governance_hit=true, exit_code=0   # 之前: exit_code=2 误报
```

**设计取舍**:

- 阈值 2 而非 1:防止单条 README 引用 `99-SKILLS治理` 误触豁免
- 不影响完成度自评:治理任务声称完成时仍要 5 维自验
- stats 保留 `governance_hit` 字段:2 周后回看治理任务比例,判断阈值是否要调
