# 完成度自评规约 (Completion Self-Score Spec)

每次声明"完成 / done / 交付"前,必须按本规约自评。这是 **Claude 自己用的 quality gate**,不是给用户看的。

Stop hook 检测到完成 marker 但没看到验证 marker 时会输出提醒(见 [post_task_reminder.py](../tools/post_task_reminder.py))。

## 为什么需要

历史教训:

| 版本 | 跳过的维度 | 后果 |
|---|---|---|
| v0.3.6 | 代码层 + 集成层 | 8 个新脚本里 6 个完全没跑过,`get_field()` 字段解析没单测,CI yaml 没本地校验 |
| v0.3.7 | 全部 5 维 | 用户两次追问后自读发现 5 个真实问题: 触发词漏(`rev-dex-dumper` 缺梆梆/爱加密)、跨平台命令错(Step 5 Unix only)、内部不一致("五阶段" vs 六阶段)、README 标题断裂、06 标题陈旧 |

模式: **写完就说完成,几乎不做端到端验证**。本规约把验证强制化。

---

## 强制 5 维自评

每次声明完成前,5 维全打勾。打不上的列出来。

### 1. 代码层 (Code)

- [ ] 主路径端到端跑过一次 (不只是 lint / 静态检查)
- [ ] 改动的脚本独立跑过 (至少 dry-run)
- [ ] 新加的依赖在干净环境验证过 / 或文档明示"未在干净环境验证"

**反例**: v0.3.6 写了 cloak_recorder.py / snapshot_replay.py / snapshot_diff.py / consistency_report.py / recording_to_fixtures.py 一个都没跑过。

### 2. 文档层 (Docs)

- [ ] 自己写的所有文件至少完整读过一遍
- [ ] 代码块里的命令**至少抽查一条**实际跑通
- [ ] 表格 / 列表的对齐没断 / 标题层级没乱

**反例**: v0.3.7 写完 USAGE/INSTALL/TRIGGERS/README 共 ~600 行,声称完成时**一遍都没读**。

### 3. 集成层 (Integration)

- [ ] 内部链接 `./X.md` 目标存在
- [ ] 跨文件引用一致 (如 USAGE 提"五阶段" vs 06 文件实际写的阶段数)
- [ ] 改完一个文件后,引用它的其他文件没失效
- [ ] 关键词 / 触发词在所属 SKILL.md 的 description 里真的存在

**反例**: v0.3.7 TRIGGERS 列了"360 加固 / 梆梆 / 爱加密 / 腾讯乐固"但 rev-dex-dumper SKILL.md 里一个都没写,用户用这些词不一定触发。

### 4. 回归层 (Regression)

- [ ] 改完核心脚本 (如 `score_skills.py`) 跑过所有依赖层 (1/2/3/5)
- [ ] 之前能跑通的命令现在还能跑
- [ ] CI yaml 改完跑过 `yamllint` 或对照过 schema

**反例**: 无大问题,但 v0.3.6 改 `score_skills.py` 时只跑了 1 层,2/3/5 层是后来用户问才跑的。

### 5. 诚实层 (Honesty)

- [ ] 列出"做了什么"
- [ ] 列出"没验证什么"
- [ ] 不把"代码写完"等同于"功能能用"
- [ ] 不把"评分高"等同于"产出对"

**反例**: v0.3.6 主动汇报"42.47"是好的(诚实化),但 v0.3.7 完全没说"零验证"。

## 完成度评分公式

| 5 维全勾 | 跳过 1 维 | 跳过 2 维 | 跳过 3 维 | 跳过 4-5 维 |
|---|---|---|---|---|
| 10/10 | 7/10 | 5/10 | 3/10 | ≤2/10 |

不允许"宣称完成 → 实际分 < 7"。低于 7 必须列具体跳过项 + 解释为什么不能现在做(如外部依赖未到位)。

## 验证 Marker (Stop hook 用)

Stop hook 在 transcript 里扫两组关键词。

**完成 marker** (任一命中 = 在说"我做完了"):
```
完成 / 做完 / 交付 / 提交 / 收尾 / 结束 / 完工 / 全部完成 / 已经做完 /
done / delivered / finished / wrapped up / shipped
```

**验证 marker** (任一命中 = Claude 真的验过):
```
验证 / verified / verify / 跑过 / 实测 / 自评 / 自检 / 自读 / 端到端 /
checklist / 5 维 / 五维 / spot check / smoke test / dry-run pass
```

**触发条件**: 命中 ≥ 1 个完成 marker **且** 未命中任何验证 marker → 输出验证提醒。

提醒内容:
```
[my_reverse_skill] 完成度自评提醒:
你说"完成"了,但 transcript 里没看到验证动作。按 08-完成度自评.md 走 5 维:
  1) 代码层: 端到端跑过? 2) 文档层: 自读过? 3) 集成层: 内链查过?
  4) 回归层: 依赖层都跑了? 5) 诚实层: 列了"没验证什么"?
全打勾 = 10/10; 跳一维 = -3 分。
```

## 与其他规约的关系

| 规约 | 作用域 |
|---|---|
| `04-新增SKILL评分回测准入.md` | 新 Skill 准入门槛 |
| `07-一致性验证规约.md` | 数据层验证(我方产出 vs 真实网页) |
| **`08-完成度自评.md`(本文件)** | 工作流验证(Claude 自评是否真完成) |

三者互补: 04 管准入, 07 管数据正确性, 08 管交付质量。
