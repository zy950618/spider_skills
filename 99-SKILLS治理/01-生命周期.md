# SKILLS 生命周期

SKILLS 不是一次性提示词，而是一套可持续进化的能力系统。

## 生命周期

```text
捕获真实任务
  -> 判断是否已有 Skill
  -> 执行并记录结果
  -> 发现失败点
  -> 更新 Skill
  -> 补 eval
  -> 评分
  -> 版本升级
  -> 定期漂移测试
```

## 每次任务结束后的复盘问题

- 这次任务是否命中了正确 Skill？
- 有没有该触发但没触发？
- 有没有不该触发却触发？
- 失败属于业务、加密、路由、WAF、IP、支付风险还是代码实现？
- 哪些经验应该进 `references/`？
- 哪些失败应该进 `evals/`？
- 是否需要修改 `description`？

## 版本规则

- `0.1.0`：初版可用。
- `0.2.0`：新增关键流程、分类、重要 reference。
- `0.2.1`：修触发词、小范围 eval 或文档。
- `1.0.0`：经过多个真实任务、负例测试、漂移测试后稳定。

## 不做什么

- 不把所有经验塞进一个超长 SKILL。
- 不把编码代理规则混入逆向 Skill。
- 不把一次失败硬编码成只适配一个站点的规则。
- 不用“评分高”代替真实任务成功。
